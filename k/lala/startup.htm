Steps to begin each new Lala letters year:

  1. change the year in global.asa (intYear variable)
  2. add any new lalas at this page
  3. modify SQL view "v2" so intYear is current year, and to make sure data looks good
  4. open LalaMerge.docx (get copy here if needed), will use view v2 as merge source for lalas. Trouble with this? Within SQL admin, save v2 to CSV, then as Excel, replace all smart quote characters, then point LalaMerge.docx file to use XLSX as recipient source.
  5. print preview to ensure they all look good, then print
  6. run SQL spLalaAddNewYear (from SQL dashboard Query Analyzer) which adds all those lalas into tblLalaLocation

Adding new prefixes? (To generate even more lala combinations)

Add codes to tblLalaRoot, then re-run spLalaAddNewYear; it won't replace anything, will only add new lalas
